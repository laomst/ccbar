# Story-05: 项目级配置支持 - 需求分析文档

## 1. 需求背景

### 1.1 当前实现

当前 CCBar 的配置为**系统级**（应用级），存储位置：
- **存储路径**：`~/Library/Application Support/JetBrains/<IDE版本>/ccbar.xml`（macOS）
- **实现类**：`CCBarSettings`（`PersistentStateComponent`，应用级服务）
- **作用范围**：所有项目共享同一套配置

相关代码：
```kotlin
// CCBarSettings.kt
@State(
    name = "site.laomst.ccbar.settings.CCBarSettings",
    storages = [Storage("ccbar.xml")]
)
class CCBarSettings : PersistentStateComponent<CCBarSettings.State>
```

Toolbar 渲染逻辑：
```kotlin
// CCBarToolbarActionGroup.kt
override fun getChildren(e: AnActionEvent?): Array<AnAction> {
    val settings = CCBarSettings.getInstance()  // 只读取系统级配置
    // ...
}
```

### 1.2 用户需求

需要增加**项目级配置**能力，具体需求：

1. **设置页面标签页**：分为"系统配置"和"项目配置"两个标签页
2. **项目配置启用**：默认无项目配置，点击"启用"按钮后复制系统配置作为初始值
3. **项目配置禁用**：不删除数据，仅标记为禁用状态
4. **项目配置重置**：重新复制系统配置覆盖当前项目配置
5. **Toolbar 渲染**：有启用的项目配置时渲染项目配置，否则渲染系统配置
6. **数据独立性**：系统配置与项目配置完全独立，启用后互不影响

### 2.2 交互行为

#### 2.2.1 项目配置状态管理

| 操作 | 触发条件 | 行为 |
|------|----------|------|
| 启用项目配置 | 点击"启用项目配置"按钮 | 复制系统配置到项目配置，标记为启用 |
| 禁用项目配置 | 点击头部"禁用项目配置"按钮 | 标记为禁用（不删除数据） |
| 重置项目配置 | 点击项目配置 Tab 内"重置为系统配置"按钮 | 重新复制系统配置覆盖项目配置 |

#### 2.2.2 设置页面 Tab 切换

| 状态 | 默认显示 Tab | 说明 |
|------|-------------|------|
| 未启用项目配置 | 系统配置（唯一 Tab） | 项目配置区域显示启用提示 |
| 已启用项目配置 | 项目配置 | 两个 Tab 均可切换 |

#### 2.2.3 Toolbar 渲染逻辑

| 项目配置状态 | Toolbar 渲染 |
|-------------|-------------|
| 无项目配置 | 系统配置 |
| 有项目配置但禁用 | 系统配置 |
| 有项目配置且启用 | 项目配置 |

### 2.3 数据存储

#### 2.3.1 存储位置

| 配置级别 | 存储位置 | 作用域 |
|---------|---------|--------|
| 系统配置 | `~/Library/.../JetBrains/<IDE>/ccbar.xml` | 全局，所有项目共享 |
| 项目配置 | `<project>/.idea/ccbar.xml` | 仅当前项目 |

#### 2.3.2 项目配置数据结构

复用现有数据结构，增加启用状态字段：

```kotlin
// 项目级配置 State
data class ProjectState(
    var enabled: Boolean = false,  // 是否启用
    var commandBars: MutableList<CommandBarConfig> = mutableListOf()  // 复用现有结构
)
```

### 2.4 视觉规范

| 元素 | 样式 |
|------|------|
| Tab 标签页 | 使用 `JBTabbedPane` 或 Kotlin UI DSL 的 tab 组件 |
| 启用提示区域 | 使用 `JBTextArea` 或 `JEditorPane`，带 info 图标 |
| 操作按钮 | 使用标准 `JButton`，主要操作使用 `@PrimaryAction` |

## 3. 技术方案

### 3.1 UI 组件选型

| 组件 | 推荐实现 | 说明 |
|------|----------|------|
| Tab 切换 | `TabbedPane` (Kotlin UI DSL) | 支持两个配置面板切换 |
| 启用提示区 | `Banner` (Kotlin UI DSL) | 带图标和按钮的信息提示 |
| 操作按钮区 | `Row` + `Button` | 启用/禁用/重置按钮 |

### 3.2 状态管理

需要新增的类：

```kotlin
/**
 * 项目级配置管理类
 * 存储在 .idea/ccbar.xml
 */
@State(
    name = "site.laomst.ccbar.settings.CCBarProjectSettings",
    storages = [Storage(StoragePathMacros.WORKSPACE_FILE)]
)
class CCBarProjectSettings : PersistentStateComponent<CCBarProjectSettings.ProjectState> {

    data class ProjectState(
        var enabled: Boolean = false,
        var commandBars: MutableList<CommandBarConfig> = mutableListOf()
    )

    // ... 实现略

    companion object {
        fun getInstance(project: Project): CCBarProjectSettings =
            project.getService(CCBarProjectSettings::class.java)
    }
}
```

### 3.3 配置读取逻辑

修改 `CCBarToolbarActionGroup` 的配置读取逻辑：

```kotlin
override fun getChildren(e: AnActionEvent?): Array<AnAction> {
    val project = e?.project

    // 优先使用项目配置
    val buttons = if (project != null) {
        val projectSettings = CCBarProjectSettings.getInstance(project)
        if (projectSettings.state.enabled) {
            projectSettings.state.commandBars
        } else {
            CCBarSettings.getInstance().state.commandBars
        }
    } else {
        CCBarSettings.getInstance().state.commandBars
    }

    // ... 生成 Action
}
```

### 3.4 设置页面改造

```kotlin
class CCBarSettingsPanel(private val project: Project?) {

    fun createPanel(): JComponent {
        return panel {
            // 项目级配置提示/操作区
            if (project != null) {
                projectConfigSection()
            }

            // Tab 或直接显示配置面板
            tabs {
                tab("系统配置") {
                    // 现有配置面板
                }
                // 仅在启用项目配置时显示
                if (projectConfigEnabled) {
                    tab("项目配置") {
                        // 项目配置面板
                    }
                }
            }
        }
    }
}
```

## 4. 影响范围

### 4.1 需修改的文件

| 文件 | 修改内容 |
|------|----------|
| `CCBarSettings.kt` | 无需修改（保持系统级配置不变） |
| `CCBarSettingsConfigurable.kt` | 传入 Project 参数，支持项目级配置 |
| `CCBarSettingsPanel.kt` | 增加 Tab 切换、项目配置启用/禁用/重置功能 |
| `CCBarToolbarActionGroup.kt` | 修改配置读取逻辑，优先使用项目配置 |
| `CCBarCommandBarAction.kt` | 可能需要适配配置来源 |

### 4.2 需新增的文件

| 文件 | 内容 |
|------|------|
| `CCBarProjectSettings.kt` | 项目级配置管理类 |

### 4.3 不受影响的部分

- 配置数据结构（`CommandBarConfig`, `CommandConfig`, `QuickParamConfig`）保持不变
- 终端创建逻辑
- 弹出菜单
- 命令预览对话框

## 6. 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 项目配置文件冲突（多人协作） | 低 | 配置文件可加入 .gitignore |
| 用户对两级配置概念困惑 | 中 | 在 UI 上提供清晰的提示说明 |
| 配置迁移时数据丢失 | 低 | 启用时复制而非移动，保留系统配置 |

## 7. 后续优化建议

1. **配置导入导出**：支持项目配置的导入导出
2. **配置同步提示**：当系统配置更新时，提示是否同步到项目配置
3. **配置继承**：项目配置可设置继承系统配置的某些按钮
4. **团队共享**：支持将项目配置提交到版本控制，团队共享配置
